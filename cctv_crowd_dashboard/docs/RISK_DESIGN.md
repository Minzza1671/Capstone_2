# 위험도 산출 설계 근거

압사 위험도 파이프라인의 설계 결정과 그 근거. 발표 대비 + 향후 reconstruct 방지용.

파이프라인:
```
P2PNet(머리점) ─┐
                ├─ homography H ─ 바닥평면 좌표(m)
Farneback OF ───┘
   → 전역 면적밀도 ρ_g, kNN 국소밀도 ρ_l, 평균속도 v
   → risk_engine (Green Guide + Weidmann) → 위험등급
```

---

## 1. 모델 구성 — 왜 3개인가

| 지표 | 모델 | 역할 |
|------|------|------|
| 전역 면적밀도 ρ_g | Green Guide / Fruin | **추세 표시 전용** (대시보드). 최종등급 max엔 미포함 |
| 국소밀도 ρ_l | kNN 국소밀도 | 압사 핫스팟. 평균에 묻히는 국소 클러스터 탐지. **floor** |
| 속도 v + ρ_l | Weidmann 기본도 | 국소밀도 기준 정체 탐지 (밀도 임계 닿기 전 경보) |

**최종등급 = `max(ρ_l 등급, Weidmann 등급)`.** 밀도등급이 floor — 압사는 단일 고밀도 신호로 터지므로 가중합 희석 방지.

### ⚠️ 왜 global_level은 max에서 뺐나 (설계 정정)
`local_max ≥ global` 은 수학적 항등(국소최대 ≥ 전역평균) + density_level 단조증가 →
`local_level ≥ global_level` 항상 성립. 즉 global은 max에서 절대 못 이기는 **죽은 항**.
→ max에서 제거. 대신 ρ_g는 전역 밀도 **추세** 별도 표시로 분리(더 정교).

### ⚠️ Weidmann은 ρ_l 사용 (신호 희석 방지)
전역 ρ_g·평균 v로 돌리면 ROI 한 구석 stop-and-go가 평균에 희석됨.
→ `weidmann_level(local_max_density, mean_speed)`. 단 mean_speed는 아직 ROI 평균이라
밀도(국소)·속도(전역) **불일치 잔존** → "국소밀집 시 평균속도도 낮을 개연성 높다"는
**보수적 근사**로 방어. 정밀화(kNN 구역별 flow 분리)는 후속 과제.

원래 검토한 4모델(Green Guide/Weidmann/Helbing/Moussaïd) 중 Helbing⊂Moussaïd(둘 다 ρ×Var(v) 국소압력, 격자 적용차)라 통합. 4개 전부는 근거 중복 + 임계값 튜닝 부담 + 충돌 판단 모호. **속도분산 Var(v)는 계산만 적립(Moussaïd 국소압력 향후 확장분), 현재 등급엔 미반영.**

### 왜 두 모델(밀도+속도)을 max로 묶나 (#4 정당화)
서로 **직교하는 붕괴 모드**를 측정하기 때문:
- Green Guide 밀도 = **정적 수용한계** (얼마나 빽빽한가 → 압축 압사)
- Weidmann 속도 = **동적 흐름붕괴** (흐름이 멈췄나 → stop-and-go 압사)
둘은 다른 논문·다른 단위지만 "둘 중 **하나라도** 위험하면 위험"이 안전논리.
→ `max()` = 보수적 OR. 가중평균이 아닌 이유: 한 신호만 터져도 압사 가능(희석 금지).

## 2. 국소밀도 — 왜 kNN인가 (격자 아님)

### 격자(grid)를 버린 이유
고정 격자 셀 크기는 임의 파라미터:
- 셀 너무 큼 → 클러스터가 셀 평균에 묻힘
- 셀 = 사람 크기(~0.2㎡) → 셀당 0~1명 → 1명도 5/㎡로 양자화 폭발

→ "왜 이 셀 크기냐"에 근거 없음. **kNN은 셀 크기 파라미터 자체가 없음.**

### kNN 국소밀도 정의
각 머리점에서 k번째 최근접 이웃까지 거리 r_k (바닥평면 m):
```
ρ_i = k / (π · r_k²)   [명/㎡]
```
r_k = 실제 사람 간격 → 밀집할수록 r_k↓, ρ↑. "사람 크기 대비"가 자연히 반영됨.

### ⚠️ 출처 정확히 (발표 주의)
- **Helbing(2007)/Moussaïd(2011)의 국소밀도 = 가우시안 커널 가중**. kNN 아님.
- kNN 거리는 **ShanghaiTech/MCNN(Zhang 2016)에서 커널 폭(bandwidth) 적응 선택**에 등장.
- 안전 표현: *"kNN 국소밀도는 비모수 밀도추정 표준 계열. ShanghaiTech GT가 kNN거리로 커널폭 정하는 것과 같은 원리이며, Helbing/Moussaïd 가우시안 커널과 동일 개념(국소밀도)의 다른 추정기."*
- ❌ "Helbing/Moussaïd 방식이다"라고 하면 안 됨.

### k=8 선택 근거
합성 sweep으로 면적밀도 대비 검증:

| areal(명/㎡) | 격자 interior 배율 | Poisson(현실) 배율 |
|---|---|---|
| 0.5 | 1.27 | 0.95 |
| 1.0 | 1.27 | 0.83 |
| 2.0 | 1.27 | 1.01 |
| 4.0 | 1.27 | 1.07 |
| 6.25 | 1.27 | 0.94 |

- k=4 이하: 근접점 1~2개에 r_k→0 → 밀도 수백/㎡ 폭발 (P2PNet 중복검출에 취약).
- **k=8: 면적밀도와 스케일 정합 + 중복검출에 견고.** r_k 하한 0.3m(사람 최소간격) 클램프로 폭발 추가 차단.

### 왜 1.27 보정을 안 하는가
- 1.27은 **규칙격자 전용 artifact** (비현실적 배치).
- 실제 랜덤(Poisson) 분포 = 배율 ≈1.0 ±15%, 상수 아님.
- /1.27 하드코딩 시 현실 군중에선 오히려 과소추정 → 위험.
- 추정기는 패턴의존 편향이라 단일계수로 못 고침 → **추정기 무보정, 대신 임계값에서 안전마진.**

## 3. 임계값 — 왜 3.2/㎡인가 (안전마진)

```
FRUIN_DANGER = 4.0   # Green Guide/Fruin 위험선 (LOS D/E)
ALARM_MARGIN = 0.80  # kNN Poisson 최대 ~17% 과소추정 흡수
경보 임계     = 4.0 × 0.80 = 3.2/㎡
```

근거: **합성 Poisson sweep(우리 측정, 실데이터 아님)**에서 배율이 areal 1.0 구간 0.83
(17% 과소) — 하필 **위험 전이 구간**에서 가장 둔함. 생값 4.0 그대로면 실제 4.0/㎡일 때
kNN이 3.3~3.7 읽고 침묵.

정직한 한계 (발표 방어):
- 출처 = 합성 Poisson, **현장 실측 아님.** 실데이터 보정은 향후.
- 과소율은 **밀도의존**(전 구간 일정 X)인데 margin은 **전 구간 상수 0.80** → 의도적 단순화.
- 0.80 = 측정 최악치 0.83보다 **내림한 보수적 라운드값** (안전측). "정확 보정"이 아니라
  "측정 최악 과소추정 이상으로 잡은 안전마진"으로 설명. 매직넘버 아님 — sweep 표가 근거.

전체 등급(문헌값 × 0.80):

| 등급 | 문헌 생값 | 경보 임계 |
|------|----------|----------|
| CAUTION | 2.0 | 1.6 |
| WARNING | 3.0 | 2.4 |
| DANGER | 4.0 | 3.2 |
| CRITICAL | 5.0 | 4.0 |

코드: `risk_engine.py` `DENSITY_REF` × `ALARM_MARGIN`. 생값/마진 분리 → 발표 시 즉답 가능.

## 4. 알려진 한계 (limitation 슬라이드용)

### 경계 kNN 과소추정 (미해결)
- ROI 가장자리 점은 이웃이 안쪽에만 → r_k 부풀고 밀도 과소.
- **압사는 벽/게이트(경계)에서 먼저 터짐(Helbing)** → 추정기 최약점이 최위험 지점과 겹침.
- sweep에서 interior 1.27 vs 전체 1.04 = 경계편향 확증.
- 부분 완화: 전역 면적밀도(경계편향 없음) 교차검증.
- 완전 보정(reflection / kNN disc의 ROI 클립 면적보정)은 **향후 과제.**
- 주의: "max는 interior 지배라 영향 적음"은 **Poisson synthetic에서만 참.** 현실은 경계가 max일 수 있음.

### n < k+1 사각지대 (처리됨)
- ROI 내 9명 미만이면 kNN 무의미(이웃이 멀리서 잡힘).
- 처리: 전역 면적밀도로 fallback + `local_reliable=False` 플래그 (`build_snapshot`).

### 검출 자체 한계
- 저밀도 출력(예 0.02/㎡)은 "정확히 낮음"이 아니라 P2PNet 미검출 + kNN 희소 비신뢰 복합. 낮은 출력 ≠ 검증된 정확도.
